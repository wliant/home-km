## 3. Routing Table

All routes inside an `<AuthGuard>` component that redirects to `/login` if no valid JWT. Admin routes additionally check `user.isAdmin`.

| Path | Component | Auth | Admin Only |
|------|-----------|------|-----------|
| `/` | Redirect → `/folders` | Yes | No |
| `/login` | `LoginPage` | No | No |
| `/register` | `RegisterPage` | No | No |
| `/folders` | `FolderPage` (root view) | Yes | No |
| `/folders/:id` | `FolderPage` (folder view) | Yes | No |
| `/notes/:id` | `NoteDetailPage` | Yes | No |
| `/files/:id` | `FileDetailPage` | Yes | No |
| `/search` | `SearchPage` | Yes | No |
| `/admin/users` | `UserManagementPage` | Yes | Yes |
| `/settings` | `SettingsPage` | Yes | No |

## 4. State Management

### Zustand — `useAuthStore`

```typescript
interface AuthStore {
  token: string | null;
  user: User | null;
  setAuth: (token: string, user: User) => void;
  clearAuth: () => void;
  isAuthenticated: () => boolean;
}
```

Persisted to `localStorage` via Zustand `persist` middleware. On app init, hydrate from localStorage and validate token expiry (decode JWT `exp` claim; clear if expired).

### TanStack Query — server state

All API data flows through TanStack Query. Query keys defined as constants in `api/queryKeys.ts`:

```typescript
export const queryKeys = {
  folders: {
    tree: ['folders', 'tree'] as const,
    detail: (id: number) => ['folders', id] as const,
  },
  notes: {
    list: (params: NoteListParams) => ['notes', 'list', params] as const,
    detail: (id: number) => ['notes', id] as const,
  },
  files: {
    list: (params: FileListParams) => ['files', 'list', params] as const,
    detail: (id: number) => ['files', id] as const,
  },
  search: (params: SearchParams) => ['search', params] as const,
  tags: {
    all: ['tags'] as const,
    search: (q: string) => ['tags', 'search', q] as const,
  },
};
```

### Local state

`useState` for: form inputs, modal open/close, active tab, drag state for checklist reordering.

## 5. API Client

`src/api/client.ts`:

```typescript
const apiClient = axios.create({
  baseURL: '/api',
});

// Request interceptor — attach JWT
apiClient.interceptors.request.use(config => {
  const token = useAuthStore.getState().token;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Response interceptor — handle expired token
apiClient.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.data?.code === 'TOKEN_EXPIRED') {
      useAuthStore.getState().clearAuth();
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
```

## 6. Offline Queue UI

`useOfflineQueue` hook:
- Opens IndexedDB `homekm-offline-queue` via `idb`
- Returns `{ pendingItems, failedItems, pendingCount, failedCount }`
- Polls IndexedDB on a 5-second interval (or subscribes to IDB change events if available)

**UI integration:**
- `OfflineBanner` shown when `isOnline = false` (from `useConnectivity`)
- `FileUploadButton` shows badge with `pendingCount` when > 0
- Upload drawer lists all queued items with status indicators:
  - `pending` / `uploading`: spinner
  - `done`: checkmark (briefly shown, then removed)
  - `failed`: red warning icon + "Retry" button

## 7. Responsive Layout

### Mobile (< 768px)

- **Navigation:** Bottom tab bar with tabs: Folders, Search, Upload (action button), Settings
- **Folder tree:** Full-screen drawer, triggered by tapping current folder name in header
- **File grid:** Single-column list
- **Note list:** Single-column cards
- **Checklist items:** Large touch targets — minimum 44px height (WCAG 2.1 AA)

### Desktop (≥ 768px)

- **Navigation:** Left sidebar rail with icon + label
- **Folder tree:** Collapsible sidebar panel (left side)
- **File grid:** 3-column grid
- **Note list:** 2-column grid

### Minimum supported viewport: 320px width

## 8. Shopping List Optimisation

When viewing a note with `label = "shopping_list"`, apply a special rendering mode:

- Checklist items displayed in large font (min 18px / `text-lg`)
- High contrast: dark text on white or white text on dark background
- Checked items strike-through with reduced opacity
- "Add item" input always visible at the bottom without scrolling
- Compact layout to maximise visible items without scrolling

## 9. Child Account UI

When `user.isChild = true`:

- Header background uses a muted/desaturated version of `APP_THEME_COLOR`
- Header badge: "Kid Mode" chip
- Edit, Delete, and admin action buttons are hidden
- `/admin/users` route is not rendered (guard redirects to `/folders`)
- Push notification settings are hidden (children are not reminder recipients unless an adult adds them)
- Cannot access tag management

## 10. Markdown Rendering

Note body is stored as Markdown and rendered with:

```typescript
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm';

<ReactMarkdown remarkPlugins={[remarkGfm]}>{note.body}</ReactMarkdown>
```

GFM plugin enables: tables, task lists (Markdown syntax `- [ ]`), strikethrough, autolinks.

**Note:** Markdown task list syntax (`- [ ]`) in the body is separate from the structured `checklistItems` sub-resource. Markdown task lists render visually but do not update the `checklist_items` table.

The editor in v1 is a plain `<textarea>`. No WYSIWYG editor.

## 12. TypeScript Interface Conventions

All API response shapes have corresponding TypeScript interfaces in `src/types/api.ts`. Names mirror the spec response shapes:

```typescript
interface User { id: number; email: string; displayName: string; isAdmin: boolean; isChild: boolean; }
interface Folder { id: number; name: string; description: string | null; parentId: number | null; ownerId: number; ownerDisplayName: string; isChildSafe: boolean; createdAt: string; updatedAt: string; tags: TagSummary[]; }
interface Note { id: number; folderId: number | null; folderName: string | null; ownerId: number; ownerDisplayName: string; title: string; body: string | null; label: NoteLabel; isChildSafe: boolean; createdAt: string; updatedAt: string; checklistItems: ChecklistItem[]; reminders: Reminder[]; tags: TagSummary[]; }
interface ChecklistItem { id: number; noteId: number; text: string; isChecked: boolean; sortOrder: number; }
interface Reminder { id: number; remindAt: string; recurrence: 'daily' | 'weekly' | 'monthly' | 'yearly' | null; pushSent: boolean; recipients: ReminderRecipient[]; }
interface FileMetadata { id: number; folderId: number | null; folderName: string | null; ownerId: number; ownerDisplayName: string; filename: string; mimeType: string; sizeBytes: number; description: string | null; isChildSafe: boolean; thumbnailUrl: string | null; uploadedAt: string; updatedAt: string; tags: TagSummary[]; }
interface Tag { id: number; name: string; color: string; createdAt: string; }
interface TagSummary { id: number; name: string; color: string; }
type NoteLabel = 'recipe' | 'todo' | 'reminder' | 'shopping_list' | 'home_items' | 'usage_manual' | 'goal' | 'aspiration' | 'wish_list' | 'travel_log' | 'custom';
```
